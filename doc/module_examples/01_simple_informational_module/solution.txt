This is solution text file, which contains detailed information that will
be part of the report file. You can put here just informational message, like
this one, instructions to resolve troubles, etc.

The file can be empty and text can be generated by the check script itself.
Usually, you put here information that is static and you don't want
to generate it by the script each run again and again.

NOTE: In case that the module result is PASS (module ends by exit_pass), the
content of solution file is ignored.

For example, this file could contains such text:

* A new system which you want to use, contains many interesting features,
which could make your work much more effective. See the KnowledgeBase article
for full info: http://example.com/the-most-beautiful-features.
